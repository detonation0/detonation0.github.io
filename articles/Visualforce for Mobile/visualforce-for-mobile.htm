Visualforce for Mobile

Posted on February 4th, 2014

Creating a Mobile Visualforce Page

Because Visualforce is basically straight HTML, you can really use any form of Mobile Framework you wish. You could even build your own if you wanted! Salesforce also provides a few different options for you to take advantage of, one not requiring any knowledge of code. Here are a few examples of what you can do.

Salesforce Mobile Components

There are a few ways to create a mobile page in Visualforce. There great documentation on Salesforce’s website that outlines how to install the Mobile Components and use jQuery Mobile to display in a mobile fashion. The link for said documentation is here.

The results look pretty good.

The Mobile Components are great because you can use the component tag to write code. Setup > Develop > Components > New lets you create a new component and enter the code for it. You could write all your header code in here and save yourself from writing it on every page. For example, instead of copying the header code to every web page you can just use: <code><c:Header /></code>. That component will pull the code for the header that you wrote earlier and display it on the page, minimizing the number of lines of code on your page. The limitations of designing a site this way is you have to custom tailor a mobile page for each desktop page that you create, therefore less responsive.

Salesforce1 and SalesforceA

Salesforce1 and SalesforceA are mobile applications designed for iOS and Android. Salesforce1 is for salesmen and customers to use for viewing information. SalesforceA is for Salesforce Admins to manage their organization from anywhere by changing permission sets or managing users. These apps allow you to view data and information about your objects and custom created apps inside Salesforce. You can even create mobile layouts for your objects and view your dashboards on the go. These apps are built upon the older Chatter Mobile app and allow you to also access your Chatter feeds and customers. Examples include:

You can choose what custom tabs or apps are displayed but cannot choose the colors of the application. It’s mainly just for accessing your data, not much customization happening. But it’s all displayed in a nice looking fashion and well organized and you don’t have to know any code to use it. All of the options are available to you from Setup > Mobile Administration > Salesforce1.

Other Options

The way I’ve gone about styling my communities site for mobile is by utilizing Twitter’s Bootstrap Framework. It’s well documented and very easy to set up. What’s great is you don’t have to rely on a separate page designed specifically for mobile like with the Mobile Components and jQuery Mobile. A page created with Bootstrap will automatically respond to the screen size of the device. So no matter if you’re viewing the site on a 26” monitor or on your skinny iPhone 5, the site will resize and re-order it’s content to fit perfectly on the screen. Head over to Bootstrap’s website and see for yourself, grab the corner of the browser window and make it smaller.
